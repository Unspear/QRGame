Game

A game has 3 components:

  • Info - The game's metadata, currently only title and description.
  • Level - Defines the layout of the level, there is one level per game.
  • Code - Defines the game's logic and mechanics.

Graphics - Color

There are 8 base color IDs:

  • 0 - White
  • 1 - Grey
  • 2 - Red
  • 3 - Orange
  • 4 - Yellow
  • 5 - Green
  • 6 - Blue
  • 7 - Purple

The next 8 IDs (8 to 15) are the same colors but with the foreground and background swapped

Graphics - Characters

All graphics in this game engine are composed from Unicode text characters, custom sprites are not supported.

The game engine supports around 3300 different characters, focusing on Emoji, abstract symbols and the Latin script.

Multi-character Emoji are not supported, these include: national flags, combining diacritics, and Emoji which use gender, skin-tone or hair colour modifiers.

Code - Constants

FRAME_TIME

Frame time is a constant which is the number of seconds between each frame/update.

The game engine runs at a fixed frame rate of 60, so FRAME_TIME is always 1/60 = 0.0166…

Code - Entities

createEntity(char, color, x, y)

Creates a new entity and adds it to the world

Use createScreenEntity(...) to create an entity that is attached to the screen, which is perfect for UI elements

  • char - The string of characters which will be the appearance of the entity's sprite
  • color - The color ID of the entity's sprite
  • x
    • - The x position of the entity
  • y
    • - The y position of the entity

destroyEntity(target)

Destroys an entity, removing it from the world. Afterward, the entity object will still exist in memory and its parameters can be accessed or it could be used as a source for copyEntity(...)

  • target - The entity that will be destroyed

copyEntity(source)

Creates a new entity with the same parameters as source

  • source - The entity that will be copied
