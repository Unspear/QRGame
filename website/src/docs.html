Game

A game has three components:

  • Info - The game's metadata (title and description).
  • Level - Defines the layout of the level, there is one level per game.
  • Code - Defines the game's logic and mechanics.
Info

Metadata associated with a game:

  • Title - Displayed at the top of the player, make it descriptive but not too long.
  • Description - Displayed in the pause menu, can be longer than the title but still should not be too long.
Graphics - Colour

All sprites and tiles can have either a coloured foreground or a coloured background

When setting a sprites colour in coce, the IDs 0 to 7 cause a coloured foreground and 8 to 15 cause a coloured background

Colour IDs (foreground/background):

  • 0/8 - White
  • 1/9 - Grey
  • 2/10 - Red
  • 3/11 - Orange
  • 4/12 - Yellow
  • 5/13 - Green
  • 6/14 - Blue
  • 7/15 - Purple
Graphics - Characters

All graphics in this game engine are made from Unicode characters, custom sprites are not supported.

The game engine supports around 3300 different Unicode characters, focusing on Emoji, abstract symbols and the Latin script.

Multi-character Emoji are not supported, these include: national flags, combining diacritics, and Emoji which use gender, skin-tone or hair colour modifiers.

Level - Basics

Levels have patches, a layout, markers, and collision.

Patches are groups of tiles, they are assembled in a layout to create the tilemap.

Markers are used on top of the layout to define locations which can be referenced in code, such as spawn locations for enemies or collectables.

See the platformer demo to see an example of how these systems can be used together.

Level - Settings

The level settings define the shape of patches, the number of patches, and the shape of the layout.

When "Apply" is pressed the level will be updated with whatever settings have been entered. If the layout or patches is made smaller, or the patch count decreased, the extra data will be removed.

Level - Patches

Patches are groups of tiles of a predefined size, that can be arranged in a layout to make the level. In comparison to conventional tile-maps, this system allows for larger levels to be made more quickly and with lower storage requirements.

To edit a patch, set the "Brush" field to the character(s) you want to draw with, set the colour on the drop down, and toggle between foreground or background colouring. If you want to draw a solid colour: leave the "Brush" field empty, set the desired colour, and set it to background.

To set the brush settings to match an existing tile, tap the ⛏ icon (pick tile) an then tap the tile you want to copy.

Level - Markers

Markers are used to specify locations in a level, independent of the level layout. For example, in the platformer demo, a marker with ID "g" is used to specify the spawn locations of the geese enemies.

Markers don't do anything on there own, they must be accessed from code, see Code - Markers

Level - Collision

The collision tab determines which characters in the level will have solid collision in the physics simulation. Simply add all the characters that should have collision to the "Solid Characters" text input.

This does not affect whether entities have collision or not, for entity collision see Code - Physics

Code - Basics
Code - Constants

DELTA_TIME

Delta time is a numeric constant which is the number of seconds between each call of onUpdate.

The game engine runs at a fixed frame rate of 60, so DELTA_TIME is always 1/60 = 0.0166…

Code - Entities

Entity

Entities are the fundamental game object in the engine. Entities support graphics, physics, and input functionality.

  • pos - The position of the Entity {x, y}
  • onUpdate - An event that will be called every update
  • sprite - The sprite component, see the Code - Sprite tab for more
  • physics - The physics component, see the Code - Physics tab for more
  • input - The input component, see the Code - Input tab for more

createEntity(char, colour, x, y) → Entity

Creates a new entity and adds it to the world

Use createScreenEntity(...) to create an entity that is attached to the screen (ideal for HUD graphics)

  • char - The string of characters which will be the appearance of the entity's sprite
  • colour - The colour ID of the entity's sprite
  • x - The x position of the entity
  • y - The y position of the entity

destroyEntity(target) → Entity

Destroys an entity, removing it from the world. Afterward, the entity object will still exist in memory and its parameters can be accessed or it could be used as a source for copyEntity(...)

  • target - The entity that will be destroyed

copyEntity(source) → Entity

Creates a new entity with the same parameters as source

  • source - The entity that will be copied
Code - Component

Component

Every entity has a sprite, physics, and input component.

Each component type has its own fields, but they all also support the following:

  • parent - The entity that owns this component (mostly for internal use)
  • lpos - The local position of this component, relative to the parent entity
  • gpos - The global position of this component, read-only
  • pivot - Default value of {x=0,y=0} means the centre of the component will be at parent.pos
  • enabled - The sprite component is enabled by default, others must be enabled before being used
Code - Sprite

Sprite Component

Controls if/how an entity appears.

Has all the properties seen in Code - Component

  • char - The string of characters which defines the sprites appearance
  • colour - The colour ID of the sprite, see Graphics - Colour
  • wrap - If greater than 0, the sprite will render for this many tiles before going to the next line
  • compact - If true, half width characters are rendered compactly
  • fliph - If true, the sprite is flipped horizontally
  • fliph - If true, the sprite is flipped vertically
Code - Physics

Physics Component

  • vel - The velocity of the physics body
  • static - If true, the body is static, if false then the body will respond to collision and forces
  • sensor - If true, the body detects overlapping bodies but doesn't collide with them
  • ghost - If true, the body collides with the level but overlaps other entities
  • bounce - A value from 0 to 1, a value of one makes all collision perfectly elastic
  • friction - A value from 0 to 1, the amount of friction the body experiences
  • onFloor - If true, then the body is on the floor, read-only
  • overlapping - A list of all entities that are currently colliding or overlapping
  • onOverlapBegin - A function that is called when an entity starts overlapping or colliding
  • onOverlapEnd - A function that is called when an entity stops overlapping or colliding
Code - Input

Input Component

  • key - Optional, the keyboard key that can be used to trigger this input, see list of key codes
  • down - True when the input is currently pressed, read-only
  • onPress - A function that is called when the input starts being pressed
  • onRelease - A function that is called when the input stops being pressed
Code - Markers

getMarkers(markerId) → Array of {x, y}

Returns an array of all the markers that match the specified markerId

  • markerId - A string containing a single character which is the marker ID
Code - Timers

createTimer(duration) → Timer

Creates a timer object with the specified duration

  • duration - The duration of the timer in seconds

destroyTimer(timer)

Destroys the timer, it will no longer be updated and its finish function will not fire if it has not already

  • timer - The timer to be destroyed
Code - Camera

Camera

The global camera object defines what section the level is shown on screen.

The cameras postion is always limited so that it cannot go outside the level boundaries.

  • pos - The position of the camera (the top left corner of the screen)
Code - Ending

stopGame(...strings)

Ends the game and shows the entered strings on screen

For example: stopGame("You","Have","Won!") would show "You" on the first line, "Have" on the second line, and "Won!" on the third line.

  • ...args - The strings which will be shown
Audio - Basics

To create sound effects you start with a wave, generated by a function such as audio.triangle(...) then apply effects such as addGain(...) to control the volume or addLowpass(...).

The parameters of waves and effects can be changed using drive...(...) functions.

Finally once an effect is fully constructed, call output() on it to play it.

Audio - Waves

audio.triangle/square/sine/sawtooth(frequency, length) → OscillatorSoundNode

Creates a new OscillatorSoundNode with the specified parameters.

There is also a generic version of these functions which can be used as follows: audio.wave("triangle", 440, 0.5)

  • frequency - The frequency can be either a Hz value (e.g. 440) or a note string, with the note letter and the octave ("C#4")
  • length - The number of seconds that the wave will play for

audio.noise(length) → BufferSoundNode

Creates a new BufferSoundNode with the specified parameters.

It is separate from the other waves because white noise doesn't have a frequency.

  • length - The number of seconds that the wave will play for
Audio - Text to Speech

A basic text to speech system is provided, it is a part of the audio system so that affects can be applied.

audio.speech(string, length=0) → BufferSoundNode

Creates a new BufferSoundNode with the specified parameters.

Example 1: audio.speech("Hello World").output() will say "Hello World" and then stop

Example 2: audio.speech("Hello World", 5).output() will repeat "Hello World" for 5 seconds

  • string - The string that is to be spoken.
  • length - The number of seconds the speech will play for. If the length is 0 the speech will play through once, otherwise it will play as many times as is needed to fill the duration.
Audio - Music Macro Language

Music Macro Language has been used historically to enable basic text based musical composition.

For a syntax explanation, click here.

audio.mml(wave, composition) → MMLSoundNode

Creates a new MMLSoundNode with the specified parameters.

Example: audio.mml("square", "t200 o6 l8 e g > e c d g").output()

  • wave - The wave that will be used to play the music: "triangle", "sawtooth", "square", or "sine".
  • composition - A composition written using Music Macro Language syntax
